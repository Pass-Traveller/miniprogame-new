# 审核结果订阅消息模板配置方案

## 目标

解决用户提交志愿服务或荣誉获奖申报后，系统提示“订阅暂不可用”的问题。该提示的原因是小程序前端没有配置微信订阅消息模板 ID。

## 一、微信公众平台配置

1. 登录微信公众平台，进入对应小程序。
2. 进入“功能”中的“订阅消息”。
3. 在公共模板库中搜索并选用“审核结果通知”“申请结果通知”或“办理结果通知”这类模板。
4. 模板用途选择“用户提交积分申报后，管理员审核完成时通知用户审核结果”。
5. 选择模板字段时，建议保留以下字段：
   - 事项名称：例如“志愿服务积分申报”“荣誉获奖积分申报”。
   - 审核结果：例如“已通过”“已驳回”。
   - 审核时间：管理员完成审核的时间。
   - 备注说明：例如“通过积分 5 分”或“驳回原因：材料不清晰”。
6. 提交并保存后，复制微信后台生成的模板 ID。

## 二、项目环境变量配置

把微信后台复制的模板 ID 填入以下两个文件：

- `.env.development`
- `.env.production`

配置示例：

```ini
VITE_AUDIT_SUBSCRIBE_TEMPLATE_ID=这里填写微信后台生成的模板ID
VITE_SUBSCRIBE_TEMPLATE_IDS=
```

说明：

- `VITE_AUDIT_SUBSCRIBE_TEMPLATE_ID` 是审核结果通知的单模板配置，当前项目优先读取这个值。
- `VITE_SUBSCRIBE_TEMPLATE_IDS` 是兼容旧配置的多模板配置，多个模板 ID 用英文逗号分隔。
- 如果两个变量都配置了，系统会自动去重后一起传给微信订阅接口。

## 三、重新构建

环境变量是构建时注入的，修改后必须重新构建小程序：

```bash
npm run build:mp-weixin
```

然后用微信开发者工具重新打开或刷新 `dist/build/mp-weixin`。

## 四、推荐消息内容

审核通过示例：

```text
事项名称：志愿服务积分申报
审核结果：已通过
审核时间：2026-05-19 10:30
备注说明：通过积分 5 分
```

审核驳回示例：

```text
事项名称：荣誉获奖积分申报
审核结果：已驳回
审核时间：2026-05-19 10:30
备注说明：佐证材料不清晰，请重新上传
```

## 五、注意事项

- 订阅消息必须由用户点击按钮后主动触发，不能在页面加载时自动弹出。
- 用户每同意一次订阅，只能用于后续发送一次对应模板消息。
- 模板 ID 每个小程序都不同，不能使用其他小程序的模板 ID。
- 如果后续要由云函数在管理员审核完成时自动发送微信通知，云函数也需要配置相同模板 ID，并按微信后台最终字段编号组织 `thing1`、`phrase2`、`time3`、`thing4` 等数据。
